## 1. 背景与动机

### 1.1 当前问题

`web/index.html` 当前是"展示型首页"——Hero + 数据统计 + 6 张入口卡片 + 9 个特性卡片 + 技术栈 + CTA + Footer。虽然暗色金调设计感成熟，但：

- **未呈现差异化**：PRD 中"测量→设计→预算→采购→施工→结算全链路一体化"和"8 AI 智能体自主运营"两个核心理念，在 UI 上仅以静态文字呈现，看不出"AI 在自主工作"
- **依赖人工导航**：用户进入后需要自己判断去哪个页面、自己点按钮操作，本质还是"人用工具"模式
- **多端矩阵不可见**：PRD 4 端矩阵（业主/设计师/供应商/工长）在首页完全没有体现

### 1.2 设计目标

把 UI/UX 从"展示型门户"重塑为"AI 自治运营工作台"，让"AI 员工自治"理念从首页入口贯穿到具体工作界面：

- **入口即差异化**：首页呈现 4 角色分流 + 全链路时间轴，一眼传达"6 阶段一体化"
- **群聊即工作台**：登录后落地到"项目群聊"，8 AI Agent 作为群成员自主推送工作进展、触发审批、相互协作
- **群策群力**：8 AI Agent 在群里相互 @ 协调（业主可见协作链路），业主仅用自然语言交互
- **真实数据驱动**：真实接入后端 320 API + WebSocket 实时推送

### 1.3 范围

| 项目 | 是否在范围 | 说明 |
|---|---|---|
| `web/index.html` | ✅ | 重塑为 4 角色入口分流首页 |

| 供应商工作台 | ✅ | 新建：1v1 采购 Agent 会话 |

## 2. 核心设计决策

| # | 决策 | 选择 | 理由 |

| D1 | 重设计范围 | 首页 + 主业务页面（4 角色工作台） | 覆盖用户主路径，不扩散到辅助页面 |

### 5.2 顶栏

- 左：‹ 返回
- 中：群名（项目名）+ 副标题（"8 AI 群策群力 · 当前阶段 Day N"）
- 右：⋯ 设置入口
- **移除**：项目切换器、8 Agent 在线徽章、待审红点（这些信息通过消息流自然呈现）

### 5.3 消息流

#### 5.3.1 消息类型

| 类型 | 触发者 | 形态 | 示例 |
|---|---|---|---|
| 文本消息 | 业主 / Agent | 气泡（业主右、Agent 左） | "同意整改" |
| 任务卡片 | 施工 Agent | 卡片：标题 + 任务清单 + 进度 | 今日施工任务 |
| 照片消息 | 工长 / 业主 | 缩略图网格 + 说明 | 现场照片 3 张 |
| 审批卡片 | 总控 Agent | 卡片：问题 + 影响 + 建议 + 决策按钮 | 厨房插座整改 |
| 文档消息 | 设计 Agent | 文件图标 + 名称 + 大小 + 查看链接 | 厨房水电点位图_v2.dwg |
| 预算卡片 | 预算 Agent | 卡片：总预算/已支出/剩余 + 进度条 | 预算概览 |
| 比价卡片 | 采购 Agent | 卡片：N 个供应商对比 + 推荐 | 客厅地砖比价 |
| 系统通知 | 总控 Agent | 居中文本 + 分隔线 | "── 第 23 天 ──" |
| Agent 协作链路 | 任意 Agent | 消息底部小字："↳ 已 @ XX Agent" | ↳ 已 @总控 Agent 评估 |

#### 5.3.2 Agent 间协作可见性

当 Agent A 触发 Agent B 时，A 的消息底部显示：
```
↳ 已 @总控 Agent 评估影响
```
业主可点击展开查看协作链路（A → B → C → 决策）。

#### 5.3.3 自然语言调用（不用 @）

业主直接在输入框说话，LLM 意图理解自动路由：

| 业主说 | 路由到 | 响应 |
|---|---|---|
| "现在预算还剩多少" | 预算 Agent | 预算卡片 |
| "把厨房改成开放式" | 设计 Agent | 3 套方案 + 预算影响 |
| "今天的施工进度怎么样" | 施工 Agent | 任务卡片 |
| "A 供应商地砖什么时候到" | 采购 Agent | 订单状态卡片 |
| "插座整改好了吗" | 质检 Agent | 复检结果 |
| "怎么付款" | 结算 Agent | 付款节点 + 链接 |

意图识别置信度低时，总控 Agent 介入澄清："你是想问 A 预算还是 B 进度？"

### 5.4 底栏

- 左：+ 号（展开附件菜单：📷拍照 / 📁文件 / 📍位置）
- 中：输入框（占位符："说点什么…"）
- 右：🎤 语音输入
- **移除**：快捷功能条（@选择器、查看预算、查看方案等）

### 5.5 背景壁纸

- 路径：`/Users/netsong/Developer/suoke_life/assets/images/wallpaper/`
- 文件：`IMG_2640.webp` ~ `IMG_2657.webp`（18 张）
- 加载策略：
  - **默认随机推送**：每次进入工作台，从 18 张中随机选择一张作为背景（写入 sessionStorage 仅本次会话有效）
  - **用户自定义覆盖**：设置页提供 18 张缩略图选择器 + "随机模式"开关
    - 选择具体壁纸 → 写入 `localStorage.wallpaper`，后续加载优先使用
    - 选择"随机模式" → 清除 `localStorage.wallpaper`，恢复随机推送
  - **透明度叠加**：壁纸应用 `opacity: 0.15-0.25`，保证消息气泡可读性
- 复制策略：在 `web/assets/images/wallpaper/` 下创建副本，避免跨项目依赖

### 5.6 角色视角差异

**所有 4 角色看到的是同一个项目群**，群成员完全相同：8 AI 全员 + 业主 + 设计师 + 供应商 + 工长。差异仅在"登录身份决定的视角与可见操作"：

| 角色 | 群成员（完全相同） | 默认聚焦消息 | 可见操作 |
|---|---|---|---|
| 业主 | 8 AI + 业主 + 设计师 + 供应商 + 工长 | 全部（审批优先） | 审批 / 询问 / 上传照片 / 呼叫 Agent |
| 设计师 | 8 AI + 业主 + 设计师 + 供应商 + 工长 | 设计 Agent 推送 / 方案卡片 | 上传图纸 / 修改方案 / 回应业主 |
| 供应商 | 8 AI + 业主 + 设计师 + 供应商 + 工长 | 采购 Agent 询价卡片 | 报价 / 接单 / 更新物流 |
| 工长 | 8 AI + 业主 + 设计师 + 供应商 + 工长 | 施工 Agent 任务 / 质检 Agent 反馈 | 拍照上报 / 确认材料 / 验收申请 |

**关键含义**：
- 4 角色登录后看到的是同一个项目群的同一份消息流
- 消息流不因登录身份过滤——所有人都看到完整协作过程
- 角色仅决定：① 哪些消息对自己的"待办"高亮 ② 自己可执行哪些操作按钮
- 例如：审批卡片对业主显示"同意/整改"按钮，对工长显示"已知悉"

### 5.7 群聊壁纸与消息气泡的视觉关系

- 壁纸作为整页背景（fixed），消息流区域半透明叠加
- Agent 消息气泡：`rgba(18, 18, 29, 0.92)`（保留暗色卡片质感）
- 业主消息气泡：`rgba(42, 26, 10, 0.92)`（金色 accent）
- 审批卡片：`rgba(18, 18, 29, 0.95)` + `border: 1px solid #c9973b`

## 7. 数据流与 API 接入

### 7.1 真实接入 320 API

- **认证**：PASETO 令牌（项目硬约束：不用 JWT）
- **API 前缀**：`/api`（项目硬约束）
- **实时推送**：WebSocket（项目硬约束：< 3 秒跨端同步）
- **API 客户端**：封装 `fetch` + 自动刷新 PASETO

### 7.2 关键 API 端点（按消息类型）

| 消息类型 | API | 说明 |
|---|---|---|
| 项目信息 | `GET /api/projects/{id}` | 顶栏群名、阶段、Day N |
| Agent 状态 | `GET /api/agents/status?project_id={id}` | 8 Agent 在线状态 |
| 消息流 | `GET /api/messages?project_id={id}&limit=50` | 历史消息分页 |
| 推送消息 | WebSocket `ws://host/api/ws?token={paseto}` | 实时 Agent 推送 |
| 审批操作 | `POST /api/approvals/{id}/decide` | 通过/整改 |
| 自然语言 | `POST /api/agents/chat` | 业主自然语言 → Agent 路由 |
| 预算概览 | `GET /api/budgets/{project_id}/summary` | 预算卡片 |
| 比价报告 | `GET /api/procurement/quotes/{id}` | 比价卡片 |
| 施工日报 | `GET /api/construction/daily-reports/{id}` | 任务卡片 |
| 文档下载 | `GET /api/files/{id}` | 图纸/文档消息 |

### 7.3 WebSocket 消息协议

```json
{
  "type": "agent_message",
  "agent": "construction",
  "project_id": "proj_xxx",
  "message_type": "task_card",
  "payload": {
    "title": "今日任务 · 水电 Day 3/7",
    "tasks": [
      {"name": "厨房布管", "status": "done"},
      {"name": "穿线布线", "status": "in_progress"}
    ]
  },
  "timestamp": "2026-07-12T08:15:00+08:00"
}
```

### 7.4 Fallback 策略

- 后端不可达时：显示"AI Agent 暂时离线"提示，历史消息仍可浏览
- WebSocket 断线：自动重连（指数退避，最多 5 次）
- 单个 Agent 失败：不影响其他 Agent，失败 Agent 显示"工作中"状态

## 12. 风险与缓解

| 风险 | 缓解 |
|---|---|
| WebSocket 在弱网环境不稳定 | 指数退避重连 + 离线消息缓存 |
| LLM 意图识别误判 | 置信度 < 0.7 时总控 Agent 澄清 |
| 18 张壁纸跨项目依赖 | 复制到 `web/assets/images/wallpaper/`，避免跨项目引用 |
| 4 角色同页 workbench 复杂度 | 用角色字段驱动视图差异，单一页面骨架 |
| 真实 API 数据可能不全 | Fallback 到 mock 数据 + 提示"演示数据"标识 |
