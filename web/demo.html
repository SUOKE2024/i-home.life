Demo 创作思路

灵感来源 · 痛点 · 方向判断 · 关键取舍

灵感来源

中国住宅装修市场 4 万亿规模，但数字化程度极低。业主从毛坯到入住经历六个完全断裂的环节：测量（手动卷尺+草图）→ 设计（AutoCAD/SketchUp/3ds Max 碎片化）→ 选材（跑建材市场）→ 采购（人工比价）→ 施工（微信群沟通）→ 验收（凭经验无标准），预算常超 20–40%。

三项技术在过去 3 年集中突破：Apple RoomPlan（LiDAR 空间扫描）、OpenCascade.js（WASM 几何引擎）、LLM Agent 框架（AI 自主运营替代人工）。这是用 AI 重构装修行业的窗口期。

想解决的问题

痛点

传统装修的六大断裂

  • 测量靠手动卷尺+草图，耗时 60+ 分钟
  • 设计用碎片化工具，反复导入导出
  • 选材跑建材市场，信息高度不对称
  • 采购靠人工比价，效率极低
  • 施工靠微信群沟通，变更全凭口头
  • 验收凭经验无标准，纠纷频发
方案

AI 全链路一体化

  • LiDAR 空间扫描 3 分钟完成量房（20x 提效）
  • AI Agent 10 分钟出方案 vs 传统 2 天
  • 215 个物料 SKU 自动匹配 BOM
  • 12 家供应商 AI 自动询比价
  • 8 阶段标准化施工 + AI 质检
  • 全流程数据透明，预算偏差 ≤ 5%

为什么做这个方向

  • 市场空白明确：23 维度竞品对比显示，无一竞品覆盖超过 40%，AI Agent 自治运营是所有人缺失的能力
  • 技术可行性已达标：WASM 几何引擎、多模态 LLM、WebSocket 实时推送 — 三项关键依赖已成熟
  • 社会价值巨大：每 1% 行业效率提升，对应约 400 亿社会成本节约；AI 质检+透明比价可减少 70% 装修纠纷

开发心得

挑战 1：AI Agent 混合路由设计

纯 LLM 调用成本高且延迟大，纯规则匹配灵活性不足。最终采用混合架构：Orchestrator Agent 先用关键词规则快速分类意图，命中后调用对应专业 Agent；LLM 调用失败时自动 fallback 到 mock 模板。后续升级为 SSE 流式响应，前端逐 token 渲染，对话实时感大幅提升。

挑战 2：安全漏洞批量修复

安全审查发现 8 个可利用漏洞：PASETO 默认密钥可被伪造 + 16 个端点 IDOR 跨项目访问 + files.py 上传无类型限制。TRAE 帮助逐文件添加 project.owner_id 归属权校验 + 文件类型白名单 + 运行时密钥强度检查，437 个测试用例全部回归通过。

挑战 3：WebSocket 生产环境部署

21 个变更端点的 project_id 获取策略不统一。Nginx 生产环境缺少 /ws/ location 块导致 WebSocket 升级请求被 try_files 兜底返回 HTML 而非 HTTP 101。更新 nginx-ihome.conf 后统一配置了 /ws/ 代理、gzip 压缩和安全响应头。

我和 TRAE 的故事

从农业科研到智能装修平台

我叫老徐，大半辈子和土地、庄稼、畜禽打交道——农业科研、田间试验、饲料营养、行政管理，干到 2024 年退休，创立了 OPC 公司。中间，做过一段家装设计与施工管理，跟建材、工装、项目监理打交道，从量房、画图、跑工地，到管预算、盯进度、协调工长和供应商，装修这行的苦和累，我比谁都清楚。2026年，索克生活 APP 快上线了，就想把以前做装修踩过的坑，变成一套普通人也能用起来的智能装修平台。

TRAE 帮我规划了 PRD，从 F1 AR 空间测量一路排到 F40 三方协作 IM，整整 40 个功能模块。每当我不知道该先做什么后做什么的时候，它就把需求拆成一个一个可执行的模块：厨房设计器、卫生间设计器、灯光方案、定制家具、智能家居……我一个一个跟它说需求，它一个一个帮我实现。

最难的一关是集成阶段。那个时候已经有 34 个路由模块、300 多个 API 端点、69 张数据表了。每次新增一个功能，都要同步更新 models/__init__.py 和 app/main.py 这两个集成点，稍有不慎就牵一发动全身。TRAE 一个文件一个文件修——Python 的、Flutter 的、Shell 脚本的——修完还自己跑测试验证。我看着终端里"302 项测试全部通过，9 项跳过"那行绿字，突然觉得：以前觉得搞不定的事，不是因为难，是因为没人帮你一步一步走。

还有一个坎是鸿蒙适配。Flutter 官方不支持 HarmonyOS，要在华为手机上跑起来，得用 Flutter-OH 3.35.7 版本，还得配置 DevEco Studio、ohpm、签名证书……我自己折腾了两天都没搞定。TRAE 帮我查文档、改配置、写升级指南，硬是把这关过了。看着 Flutter 应用在 MatePad 上跑起来的那一刻，我对着屏幕笑出了声。

后来的事情你也看到了：一个又一个 Session，从 AR 空间测量到 VR 全景出图，从厨房动线分析到担保支付状态机，从项目骨架到 40 个模块全量交付，321 个 API 端点，302 项测试通过。Nginx 的安全头配置、proxy_pass 尾部斜杠的坑、CSP 策略的调试、响应式断点的添加、无障碍属性的补充……这些坑，TRAE 帮我一个一个填了。

创业很难吗？我想和你说：有了 TRAE，专业背景不再决定你能走多远。你只需要深耕一个领域，知道这个领域真正的痛点是什么，剩下的，它会陪你走完。

其实，只要敢想，有了 TRAE 你完全可以飞起来。
