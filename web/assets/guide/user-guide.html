01 ⌂ 平台简介

i-home.life（索克家居）是一个 AI 驱动的全链路智能家居装修建造平台， 覆盖从户型设计、预算估算、物料采购、施工管理、竣工结算到质量验收的完整装修生命周期。

平台由 9 个分工明确的 AI 智能体协同工作，通过统一的群聊工作台对外提供能力：

  • 全链路覆盖：设计 → 预算 → 采购 → 施工 → 结算 → 质检，每个环节均有专属 Agent。
  • 多模态交互：支持文本对话、语音对话、3D / VR / AR 设计可视化。
  • 实时协作：基于 WebSocket 与 SSE 流式响应，多成员可同步参与同一项目。
  • 安全可信：PASETO v4 令牌 + WebAuthn / Passkey 生物识别 + 身份证实名认证。

平台同时提供 Web 端（基于浏览器，支持桌面与移动浏览器）与 Flutter 客户端（iOS / Android），两端数据实时同步。

Agent Harness（智能体统一管理）

Agent Harness 是 v1.2.0 引入的智能体统一管理层，负责 9 个 AI Agent 的 生命周期管理、上下文注入、能力发现和监控追踪。

  • 统一调度：所有 Agent 通过 Harness 注册与启动，Orchestrator 通过 Harness 获取可用 Agent 列表并动态路由。
  • 上下文注入：Harness 将项目信息、用户偏好、历史对话统一注入 Agent 上下文，减少重复配置。
  • 执行追踪：记录每次 Agent 调用的输入、输出、耗时和 token 消耗，便于调试与优化。
  • 能力发现：Agent 通过 Harness 声明自身能力（如「支持户型设计」「支持预算估算」），供 Orchestrator 智能匹配。

对话会话管理

平台已全面支持 AI Agent 对话的自动持久化保存，确保您的对话历史和上下文不丢失。

  • 自动保存：与任意 AI Agent 的对话消息（包括用户提示词和 AI 响应）均会自动保存，无需手动操作。
  • 历史查看：Web 端点击工作台顶部的「📋 历史」按钮，移动端点击导航栏的历史图标，即可浏览过往对话记录。
  • 持久化存储：对话数据在页面刷新、App 关闭重启后依然保留，登录即可恢复之前的会话上下文。
  • 新建会话：随时可点击「新建对话」开启一个全新的会话，之前的对话历史仍然保留可供查阅。
  • 隐私保护：所有对话内容使用 Fernet 对称加密存储，仅账号持有者本人可访问，他人无法查看您的对话记录。

03 ⌨ 工作台使用

工作台是用户与 AI 智能体交互的核心界面，采用群聊模式：所有 Agent 与项目成员都在同一会话中， 通过 @提及 指定由谁响应。

群聊工作台

  • 统一会话：项目内所有沟通与 Agent 调用共享同一上下文，避免信息割裂。
  • @提及：输入 @Designer、@Budget 等可直接呼叫对应 Agent；不指定则由 Orchestrator 自动路由。
  • 历史消息：会话历史持久化存储，可向上滚动查看过往交互；移动端支持无限滚动加载。
  • 多端同步：Web 端与 Flutter 客户端通过 WebSocket 实时同步消息。

SSE 流式响应

Agent 响应采用 SSE（Server-Sent Events）真流式传输： 首个 token 通常在约 5 秒内到达，后续内容以增量方式逐字推送，无需等待完整生成。

示例：发送消息HTTP
# 发送一条 @Designer 的消息
POST /api/v1/chat/messages
Content-Type: application/json
Authorization: Bearer v4.local.xxx...

{
  "project_id": "proj_abc123",
  "content": "@Designer 帮我设计 90㎡ 两居室的方案"
}
响应：SSE 流式输出SSE
# 首个 token ~5s 到达，后续逐字增量
event: delta
data: {"delta": "根据您的需求"}

event: delta
data: {"delta": "，建议采用北欧风格..."}

event: meta
data: {"agent": "Designer", "json_schema": "design_plan_v1"}

event: done
data: {"total_tokens": 1284, "elapsed_ms": 18720}

提示：完整的 LLM 推理响应通常需要 20–40 秒完成，这是推理模型的固有特性， 请耐心等待流式输出结束，不要重复发送同一问题。

04 ▤ 项目管理

所有装修工作都以「项目」为单位组织，一个项目即一个完整的装修工程。

创建项目

  1. 进入「项目管理」→ 点击「新建项目」。
  2. 填写项目名称、地址、户型信息（面积、房型、层高）。
  3. 选择装修风格与预算范围（可选）。
  4. 提交后系统将自动初始化项目会话与默认 Agent 上下文。

户型设计

在项目内 @Designer 即可发起户型设计请求。Designer 会生成 结构化 JSON 方案（包含房间布局、面积、材料、估算造价等字段）， 同时自动提取友好回复展示给用户，方便阅读与分享。

物料清单 BOM

Designer 方案确认后，可一键调用 @Procurement 生成 BOM（Bill of Materials）：

  • 按品类汇总（主材 / 辅材 / 家电 / 软装）。
  • 关联供应商与产品 SKU，可直接发起询价与下单。
  • 支持导出为 Excel 或 JSON 格式。

预算管理

@Budget 会基于 Designer 方案与 Procurement 报价生成预算看板， 支持硬装 / 软装 / 家电 / 人工的分项拆解，并可对比计划与实际支出。

05 ◈ 设计工具

平台提供三种可视化设计工具，覆盖从方案预览到实地测量的完整链路。

3D 查看器

  • 支持 WebGPU 优先渲染（v1.2.0），在不支持 WebGPU 的设备上自动降级为 WebGL。
  • WebGPU 相比 WebGL 提供更快的 3D 渲染性能，复杂场景下帧率提升 2–3 倍。
  • 支持设计方案的全景 3D 预览，可旋转、缩放、第一人称漫游，支持材质实时替换。
  • 无需安装插件，主流浏览器原生支持。

VR 查看器

  • 支持主流 VR 头显（WebXR 兼容设备）。
  • 沉浸式漫游设计场景，1:1 真实尺度体验。
  • 移动端可使用 Cardboard 模式体验基础 VR。

AR 空间测量

AR 空间测量功能支持四级降级策略，根据设备能力自动选择最优方案：

拍照上架

供应商可通过拍照上架功能快速录入产品信息：

  • 使用设备摄像头拍摄产品照片（瓷砖、地板、涂料、灯具、家电等），系统自动裁剪压缩。
  • 多模态 AI 视觉识别（DeepSeek Vision / GLM-4V）自动分析产品名称、分类、材质、颜色、风格等属性。
  • 识别结果支持手动调整后一键创建产品，AI 自动生成营销文案。
  • 支持的产品类别：瓷砖、地板、橱柜、涂料、灯具、家电、窗帘、定制家具、服务等 9 大类。

⚠ 拍照上架仅开放给已实名认证的供应商用户使用。

智能家居摄像头

平台支持将智能摄像头（如小米、海康威视等品牌）接入智能家居系统， 实现装修现场远程巡检、安防监控和场景联动。

  • 可配置场景自动化：如「离家模式」自动开启所有摄像头录像。
  • 摄像头的设备能力包括：开始录像、停止录像、切换模式。

Sketch-to-3D 手绘识别（v1.2.0 新增）

上传手绘户型草图，AI 自动识别墙体、门窗和房间布局，并生成 3D 设计方案。

  • 上传草图：支持 PNG / JPG / JPEG 格式，最大 10MB，推荐 1024×768 以上分辨率。
  • AI 识别：自动检测墙体位置、门窗开口、房间数量和预估面积，输出结构化数据。

  • 提示：使用黑色笔在白纸上绘制，标注房间名称和尺寸，可显著提升识别准确率。

06 ⚿ 认证与安全

平台采用多重认证与加密机制保障账户与数据安全。

PASETO v4 令牌（非 JWT）

会话令牌采用 PASETO v4 协议（Platform-Agnostic Security Tokens）， 而非传统 JWT。PASETO 通过非对称签名消除了 JWT 的算法歧义风险， 令牌格式如下：

令牌结构PASETO v4
v4.public.<payload>.<signature>

# payload 为 base64 编码的 JSON：
{
  "sub": "user_xxx",
  "iat": 1752883200,
  "exp": 1752969600,
  "scope": "user"
}

令牌通过 Ed25519 私钥签名，公钥由服务端校验；过期后需重新刷新，敏感操作会强制重新认证。

WebAuthn / Passkey 生物识别

  • 支持指纹、面容、平台通行密钥（Passkey）等生物 / 硬件认证。
  • 生物特征数据由设备安全芯片（Secure Enclave / TPM）本地处理，平台仅存储不可逆的公钥凭据。
  • 支持跨设备 Passkey 同步（iOS / Android / 桌面浏览器）。

身份证实名认证

  • 仅在涉及结算、合同签署等高敏感场景要求实名认证。
  • 身份证信息使用 AES-256-GCM 加密存储，传输全程 TLS 1.2+。
  • 支持 OCR 自动识别与人工核对双校验。

07 ♪ 语音功能

平台集成 Qwen Audio 多模态语音模型，提供端到端的语音交互能力。

实时语音对话

  • 支持中文实时语音对话，端到端延迟约 1–2 秒。
  • 可与任意 Agent 进行语音交互，Orchestrator 自动路由。
  • 移动端支持后台语音模式（锁屏继续对话）。

语音转写与情绪检测

  • 实时转写：语音输入同步转写为文本，方便检索与回看。
  • 情绪检测：基于声学特征与语义内容识别用户情绪（中性 / 焦虑 / 不满 / 满意等）。
  • 情绪数据用于 Concierge（客服）Agent 的应答策略调整，必要时触发人工接管。

语音指令

支持自然语言指令，例如「创建一个新项目」「显示本周施工进度」「给设计师发消息」， 系统将解析意图并自动调用对应 Agent 或页面跳转。

建立语音会话HTTP
# 建立双向语音会话（WebSocket）
wss://api.i-home.life/v1/voice/session?token=v4.local.xxx

# 客户端推送音频帧（16kHz PCM）
{ "type": "audio", "format": "pcm_16k", "data": "<base64>" }

# 服务端推送转写 + 情绪

08 ⚑ 通知与协作

设备推送通知

  • Web 端使用 Web Push API，移动端使用 FCM / APNs。
  • 支持的推送类型：施工里程碑、验收结果、付款提醒、Agent @提及、整改通知。
  • 可在「设置 → 通知」中按类型订阅 / 取消。

WebSocket 实时通信

  • 所有在线会话通过 WebSocket 长连接保持实时同步。
  • 断线自动重连，消息按序送达（基于消息序号去重）。
  • 支持消息已读回执、输入中状态、在线状态显示。

项目成员协作

  • 一个项目可邀请多名成员（业主、设计师、施工方、监理等）。
  • 支持基于角色的权限控制（Owner / Editor / Viewer）。
  • 成员在群聊中 @成员名 可定向通知；Agent 与成员消息混合展示，统一上下文。

09 ? 常见问题

为什么 Agent 响应需要 20–40 秒？
平台使用推理模型（Reasoning Model）进行多步思考，每次响应需要经历意图理解、 上下文检索、方案生成、JSON 校验等多个阶段。20–40 秒为推理模型的固有延迟， 非网络或服务问题。SSE 流式响应可让首个 token 在约 5 秒内呈现，建议优先阅读流式输出而非等待完整响应。
什么是 SSE 真流式？和普通响应有什么区别？
普通响应需要等 Agent 完整生成后才一次性返回；SSE（Server-Sent Events）真流式 则在模型生成过程中以增量方式推送内容，首个 token 约 5 秒到达，后续内容逐字呈现。 用户可即时看到 Agent 的输出进度，体验更接近真人对话。
我的数据安全吗？生物信息会被存储吗？
所有会话令牌采用 PASETO v4 加密签名；身份证等敏感信息使用 AES-256-GCM 加密存储；传输全程 TLS 1.2+。 生物特征（指纹 / 面容）由设备本地安全芯片处理，平台仅存储不可逆的公钥凭据， 不会上传或存储您的原始生物数据。详见 隐私政策。
为什么必须用 PASETO 而不是 JWT？
JWT 存在算法歧义风险（如 alg: none 攻击、HS/RS 混淆攻击）， 而 PASETO v4 强制使用 Ed25519 非对称签名，协议层面消除了算法可选项， 安全性更高。PASETO 也是平台统一的安全令牌规范。
不指定 @Agent 时，请求会发给谁？
由 Orchestrator（总控）根据语义自动分类并路由到最合适的专业 Agent。 例如输入「帮我估个预算」会被路由到 Budget；输入「设计一下户型」会被路由到 Designer。 若需精确指定，请使用 @AgentName 显式提及。
AR 测量在我的手机上精度不够怎么办？
AR 空间测量采用四级降级策略（LiDAR → VisualSLAM → Photogrammetry → Manual）， 系统会根据设备能力自动选择。若精度不足，可尝试在光线充足、纹理丰富的环境中使用； 必要时可切换到 Manual 模式手动输入尺寸。
