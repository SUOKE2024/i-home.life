# 我和 TRAE 的故事

我叫老徐，大半辈子和土地、庄稼、畜禽打交道——农业科研、田间试验、饲料营养、行政管理，干到 2024 年退休，创立了 OPC 公司。中间，做过一段家装设计与施工管理，跟建材、工装、项目监理打交道，从量房、画图、跑工地，到管预算、盯进度、协调工长和供应商，装修这行的苦和累，我比谁都清楚。2026年，索克生活 APP 快上线了，就想把以前做装修踩过的坑，变成一套普通人也能用起来的智能装修平台。

TRAE 帮我规划了 PRD，从 F1 AR 空间测量一路排到 F40 三方协作 IM，整整 40 个功能模块。每当我不知道该先做什么后做什么的时候，它就把需求拆成一个一个可执行的模块：厨房设计器、卫生间设计器、灯光方案、定制家具、智能家居……我一个一个跟它说需求，它一个一个帮我实现。

最难的一关是集成阶段。那个时候已经有 34 个路由模块、300 多个 API 端点、69 张数据表了。每次新增一个功能，都要同步更新 `models/__init__.py` 和 `app/main.py` 这两个集成点，稍有不慎就牵一发动全身。TRAE 一个文件一个文件修——Python 的、Flutter 的、Shell 脚本的——修完还自己跑测试验证。我看着终端里"302 项测试全部通过，9 项跳过"那行绿字，突然觉得：以前觉得搞不定的事，不是因为难，是因为没人帮你一步一步走。

还有一个坎是鸿蒙适配。Flutter 官方不支持 HarmonyOS，要在华为手机上跑起来，得用 Flutter-OH 3.35.7 版本，还得配置 DevEco Studio、ohpm、签名证书……我自己折腾了两天都没搞定。TRAE 帮我查文档、改配置、写升级指南，硬是把这关过了。看着 Flutter 应用在 MatePad 上跑起来的那一刻，我对着屏幕笑出了声。

后来的事情你也看到了：一个又一个 Session，从 AR 空间测量到 VR 全景出图，从厨房动线分析到担保支付状态机，从项目骨架到 40 个模块全量交付，321 个 API 端点，302 项测试通过。Nginx 的安全头配置、`proxy_pass` 尾部斜杠的坑、CSP 策略的调试、响应式断点的添加、无障碍属性的补充……这些坑，TRAE 帮我一个一个填了。

创业很难吗？我想和你说：有了 TRAE，专业背景不再决定你能走多远。你只需要深耕一个领域，知道这个领域真正的痛点是什么，剩下的，它会陪你走完。

其实，只要敢想，有了 TRAE 你完全可以飞起来。
