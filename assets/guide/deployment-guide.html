02后端部署

后端 FastAPI 应用的部署步骤。

2.1 创建虚拟环境并安装依赖

# 克隆项目
git clone <repo-url> i-home.life
cd i-home.life

# 创建 Python 虚拟环境
python3.12 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

2.2 配置环境变量

# 复制示例配置
cp .env.example .env

# 编辑 .env，务必修改以下项：
# - PASETO_SECRET_KEY  (生成 32+ 字节随机密钥)
# - DEEPSEEK_API_KEY   (留空则使用 mock 模式)
# - AMAP_API_KEY       (高德地图，留空则禁用地址补全)
# - DEBUG=false        (生产环境设为 false)
安全警告

生产环境必须将 DEBUG 设为 false，否则 CORS 将允许所有来源（*）访问 API。同时务必更换 PASETO_SECRET_KEY 默认值。

2.3 数据库迁移

# 应用数据库迁移
source .venv/bin/activate
alembic upgrade head

# 如果数据库为空，init_db() 会自动创建表并注入种子数据（225 SKU）
# 也可手动初始化：
PYTHONPATH=. python -c "import asyncio; from app.database import init_db; asyncio.run(init_db())"

2.4 启动服务

# 开发模式（热重载）
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 生产模式（多进程）
uvicorn app.main:app --host 0.0.0.0 --port 8000 --workers 2 --log-level info

# 或使用一键部署脚本
bash scripts/deploy.sh start

# 查看状态 / 日志
bash scripts/deploy.sh status
bash scripts/deploy.sh logs
验证

启动后访问 http://localhost:8000/api/docs 查看 Swagger UI，确认所有路由已正确加载。

2.5 生产环境基础设施部署（v1.0.1 新增）

生产环境建议部署以下基础设施组件。开发环境仅需 SQLite，所有组件均为可选降级，按需启用。

PostgreSQL 部署（推荐生产环境）

生产环境推荐使用 PostgreSQL 替代 SQLite，支持更高并发与更完善的事务处理。

# 1. 安装 PostgreSQL 16+
sudo apt install postgresql postgresql-contrib

# 2. 启动并设置开机自启
sudo systemctl start postgresql
sudo systemctl enable postgresql

# 3. 创建数据库与用户
sudo -u postgres psql
CREATE DATABASE ihome;
CREATE USER ihome_user WITH ENCRYPTED PASSWORD 'your-strong-password';
GRANT ALL PRIVILEGES ON DATABASE ihome TO ihome_user;
\q

# 4. 配置 .env
DATABASE_URL=postgresql+asyncpg://ihome_user:your-strong-password@localhost:5432/ihome

# 5. 应用迁移
source .venv/bin/activate
alembic upgrade head
驱动说明

连接字符串使用 postgresql+asyncpg:// 前缀，asyncpg 为异步驱动（已在 requirements.txt 中添加）。

Redis 部署（可选）

用于缓存与消息队列。留空则自动降级为内存字典，开发环境无需安装。

# 1. 安装 Redis 7+
sudo apt install redis-server

# 2. 启动并设置开机自启
sudo systemctl start redis-server
sudo systemctl enable redis-server

# 3. 配置 .env
REDIS_URL=redis://localhost:6379/0

# 4. 验证连接
redis-cli ping
# 预期响应: PONG

OSS 对象存储配置（可选）

用于存储图纸、照片、3D 模型等大文件。支持阿里云 OSS；留空则使用本地文件存储。

# 1. 登录阿里云控制台，创建 OSS Bucket
#    - Bucket 名称: ihome-files
#    - 地域: 华东1（杭州）或就近选择
#    - 读写权限: 私有

# 2. 创建 AccessKey（建议使用 RAM 子账号）
#    - 阿里云控制台 → RAM → 用户 → 创建用户 → 附加 OSS 权限策略

# 3. 配置 .env
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_ACCESS_KEY=your-access-key-id
OSS_SECRET_KEY=your-access-key-secret
OSS_BUCKET=ihome-files
OSS_REGION=cn-hangzhou
安全提示

AccessKey 与 SecretKey 属于敏感信息，仅配置在 .env 中（已在 .gitignore 排除）。建议使用 RAM 子账号并最小化权限。

向量数据库部署（可选）

用于 RAG 知识库（装修规范、国标、产品目录检索）。支持 Qdrant 或 Milvus；留空则禁用 RAG 功能，不影响其他模块。

# ── 方案一：Qdrant（推荐，轻量级） ──
# 安装 Qdrant（二进制部署，不使用 Docker）
wget https://github.com/qdrant/qdrant/releases/download/v1.12.0/qdrant-x86_64-unknown-linux-gnu.tar.gz
tar xzf qdrant-*.tar.gz
./qdrant --config-dir config/

# 配置 .env
VECTOR_DB_URL=http://localhost:6333
VECTOR_DB_COLLECTION=ihome_knowledge

# ── 方案二：Milvus（适合大规模） ──
# 参考 Milvus 官方文档进行二进制部署
# 配置 .env
VECTOR_DB_URL=http://localhost:19530
VECTOR_DB_COLLECTION=ihome_knowledge
降级说明

向量库留空时，客服 Agent 仍可使用 FAQ 预置知识库正常工作。RAG 检索增强功能仅在配置向量库后启用。

部署优先级

必选：PostgreSQL（生产环境） | 推荐：Redis（缓存加速）、OSS（大文件存储） | 可选：向量数据库（RAG 知识库）。可按业务量逐步启用。

03Nginx 配置

Nginx 负责静态文件服务、安全头注入、反向代理、WebSocket 升级与 Gzip 压缩。

3.1 安装与启用配置

# 将配置文件放置到 Nginx 站点目录
cp scripts/nginx-ihome.conf /etc/nginx/sites-available/ihome

# 创建软链接启用
ln -s /etc/nginx/sites-available/ihome /etc/nginx/sites-enabled/

# 测试配置语法
nginx -t

# 重新加载配置
nginx -s reload

3.2 完整 Nginx 配置

生产环境使用 HTTP（IP 直接访问），避免 SSL 证书域名不匹配问题。未来启用域名+SSL 后可切换为 HTTPS。

server {
    listen 8081;
    server_name i-home.life www.i-home.life 118.31.223.213;

    # 隐藏 Nginx 版本号
    server_tokens off;

    # ── 安全响应头 ──
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;
    add_header Content-Security-Policy "default-src 'self'; script-src 'self' 'unsafe-inline' https://cdn.jsdelivr.net https://cdnjs.cloudflare.com; style-src 'self' 'unsafe-inline' https://fonts.googleapis.com; font-src 'self' https://fonts.gstatic.com; img-src 'self' data: blob:; connect-src 'self' ws: wss:; media-src 'self' blob:;" always;
    add_header Permissions-Policy "camera=(self), microphone=(self), geolocation=(self)" always;

    # ── Web 静态文件 ──
    root /opt/ihome/web;
    index index.html;

    # 旧路径重定向到 /api 前缀
    location = /docs { return 301 /api/docs; }
    location = /redoc { return 301 /api/redoc; }
    location = /openapi.json { return 301 /api/openapi.json; }

    location / {
        try_files $uri $uri/ /index.html;
    }

    # ── API 反向代理（保留 /api 前缀） ──
    location ^~ /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
    }

    # ── WebSocket ──
    location /ws/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }

    # ── 静态资源缓存 ──
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|woff2?)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # ── Gzip 压缩 ──
    gzip on;
    gzip_types text/plain text/css application/json application/javascript
               text/xml application/xml text/javascript;
    gzip_min_length 1000;
}

3.3 配置要点说明

配置项说明
server_tokens off隐藏 Nginx 版本号，减少信息泄露
安全响应头X-Frame-Options 防点击劫持、CSP 防 XSS、Referrer-Policy 控制引用来源
^~ /api/优先前缀匹配，保留 /api 前缀不剥离，确保后端路由正确
WebSocket 升级proxy_http_version 1.1 + Upgrade/Connection 头，支持 ws:// 协议
静态资源缓存30 天缓存 + immutable 标记，减少重复请求
Gzip压缩文本类响应，最小 1000 字节，降低带宽消耗

07常见部署问题排查

部署过程中可能遇到的问题与解决方案。

问题 1：启动报错 ModuleNotFoundError

原因：虚拟环境未激活或依赖未安装。

解决：

source .venv/bin/activate
pip install -r requirements.txt
# 确认 Python 版本 >= 3.12
python --version

问题 2：数据库迁移失败

原因：数据库文件权限或路径问题。

解决：

# 确认 data 目录存在且可写
mkdir -p data
chmod 755 data

# 检查迁移状态
alembic current
alembic history

# 如需重置（谨慎操作，会丢失数据）
rm data/ihome.db
alembic upgrade head

问题 3：Nginx 502 Bad Gateway

原因：后端 uvicorn 服务未启动或端口不匹配。

解决：

# 检查后端是否运行
bash scripts/deploy.sh status

# 检查端口监听
ss -tlnp | grep 8000

# 查看后端日志
bash scripts/deploy.sh logs

# 确认 Nginx proxy_pass 端口与 uvicorn 端口一致
# 默认: proxy_pass http://127.0.0.1:8000;

问题 4：WebSocket 连接失败

原因：Nginx 未配置 WebSocket 升级头。

解决：确认 Nginx 配置中 /ws/ location 包含以下指令：

location /ws/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}

问题 5：API 文档（/api/docs）返回 404

原因：FastAPI 文档路径配置或 Nginx 代理前缀问题。

解决：确认 app/main.py 中 docs_url="/api/docs"，且 Nginx location ^~ /api/ 使用 ^~ 前缀匹配以确保优先级。旧路径 /docs 会自动 301 重定向到 /api/docs。

问题 6：CORS 跨域错误

原因：DEBUG=false 时 CORS 仅允许白名单来源。

解决：在 .env 中配置 CORS_ORIGINS，添加前端域名：

CORS_ORIGINS=["http://118.31.223.213:8081","http://i-home.life:8081","http://localhost:8766"]

问题 7：鸿蒙 HAP 构建失败

原因：DevEco Studio 版本或签名配置问题。

解决：

# 运行环境诊断
bash scripts/check-ohos-env.sh

# 确认 Flutter 版本
flutter --version  # 应为 3.35.7-ohos-0.0.3

# 检查签名文件
ls flutter_app/ohos/signing/

# 参阅升级指南
cat flutter_app/ohos/OHOS_UPGRADE_GUIDE.md

问题 8：数据库 Schema 漂移（no such column）

原因：模型新增字段后，init_db() 的 create_all 只创建不存在的表，不会给已有表添加新列，导致 OperationalError: no such column。

解决：v1.0.8 起已内置轻量级迁移机制（app/database.py 的 _run_lightweight_migrations()），启动时自动检测并 ALTER TABLE。手动修复：

# 备份数据库
cp data/ihome.db data/ihome.db.bak.$(date +%Y%m%d%H%M%S)

# 添加缺失列
sqlite3 data/ihome.db "ALTER TABLE users ADD COLUMN sub_role VARCHAR(30)"
