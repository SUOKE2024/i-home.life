01技术架构

平台采用前后端分离架构，后端 FastAPI 提供 RESTful API，前端 Web SPA + Flutter 跨平台移动端。

后端

Python 3.12 + FastAPI + SQLAlchemy 2.0 (async)

异步 Web 框架，uvicorn ASGI 服务器。SQLAlchemy 2.0 异步 ORM + aiosqlite 驱动。数据库为 SQLite（Phase 1），Phase 2 切换 PostgreSQL。Pydantic v2 进行数据校验。

认证

PASETO v4 (local)

使用 PASETO（Platform-Agnostic Security Tokens）替代 JWT，对称密钥加密（SymmetricKey），令牌有效期 24 小时。比 JWT 更安全，无算法混淆攻击风险。

AI

DeepSeek LLM + 规则混合路由

7 个 AI Agent（总控/设计/预算/采购/施工/结算/质检），支持 mock 模式（无 API Key 时使用规则引擎）与 LLM 模式（接入 DeepSeek API）。意图路由自动分发到对应 Agent。

前端

Vanilla JS + Canvas 2D + Three.js r128

Web 管理后台为纯 JS 单页应用（无框架依赖），2D CAD 使用 Canvas API，3D 渲染使用 Three.js。导出格式为 DXF R12（POLYLINE）+ Excel（openpyxl）。

移动端

Flutter 3.35.7-ohos-0.0.3

跨平台 App，支持 iOS / iPadOS / Android / HarmonyOS。2D 绘图使用 CustomPainter，AR 测量通过 MethodChannel 调用原生 ARKit / ARCore / AR Engine。手写笔适配（M-Pencil）。

迁移

Alembic（SQLite / PostgreSQL 双库）

数据库迁移工具，支持 SQLite 与 PostgreSQL 双库切换。自动检测模型差异并生成迁移脚本。

技术栈一览

后端依赖详见 requirements.txt：FastAPI >= 0.136、uvicorn[standard]、SQLAlchemy[asyncio] >= 2.0.50、aiosqlite、paseto >= 2.0、cryptography、pydantic >= 2.13、alembic >= 1.18、httpx、openpyxl 等。v1.0.1 新增：asyncpg（PostgreSQL）、redis、oss2（阿里云 OSS）、qdrant-client（向量库）。

基础设施配置（v1.0.1 新增）

平台支持从开发环境（SQLite + 内存字典 + 本地文件）平滑切换至生产环境（PostgreSQL + Redis + OSS + 向量库），所有配置通过 app/config.py（pydantic-settings）统一管理，环境变量在 .env 中设置。

数据库（必选）

开发环境：默认使用 SQLite，无需额外配置。

生产环境：切换至 PostgreSQL，设置 DATABASE_URL 环境变量。驱动使用 asyncpg（异步）。

# 开发环境（默认）
DATABASE_URL=sqlite+aiosqlite:///./data/ihome.db

# 生产环境
DATABASE_URL=postgresql+asyncpg://user:pass@localhost:5432/ihome
切换说明

切换数据库后需执行 alembic upgrade head 重新应用迁移。Alembic 已配置 SQLite / PostgreSQL 双库支持。

Redis（可选）

用于缓存与消息队列。设置 REDIS_URL 启用；留空则自动降级为内存字典，开发环境无需安装 Redis。

# 启用 Redis
REDIS_URL=redis://localhost:6379/0

# 留空则降级为内存字典（开发环境推荐）
REDIS_URL=

OSS 对象存储（可选）

用于存储图纸、照片、3D 模型等大文件。支持阿里云 OSS；留空则使用本地文件存储。

# 启用阿里云 OSS
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
OSS_ACCESS_KEY=your-access-key-id
OSS_SECRET_KEY=your-access-key-secret
OSS_BUCKET=ihome-files
OSS_REGION=cn-hangzhou

# 留空则使用本地文件存储（data/uploads/）
# OSS_ENDPOINT=
安全提示

OSS Access Key 与 Secret Key 属于敏感信息，务必仅配置在 .env 中，切勿提交到版本控制。

向量数据库（可选）

用于 RAG 知识库（装修规范、国标、产品目录检索）。支持 Qdrant 与 Milvus；留空则禁用 RAG 功能，不影响其他模块。

# 启用 Qdrant
VECTOR_DB_URL=http://localhost:6333
VECTOR_DB_COLLECTION=ihome_knowledge

# 启用 Milvus
VECTOR_DB_URL=http://localhost:19530
VECTOR_DB_COLLECTION=ihome_knowledge

# 留空则禁用 RAG（客服 Agent 仍可使用 FAQ 预置知识库）
# VECTOR_DB_URL=
降级策略总结

Redis 留空 → 内存字典 | OSS 留空 → 本地文件存储 | 向量库 留空 → 禁用 RAG（FAQ 仍可用）。开发环境仅需 SQLite，零外部依赖即可运行。

04认证机制：PASETO 令牌流程

平台使用 PASETO v4 local 模式进行令牌认证，替代传统 JWT。

为什么选择 PASETO？

  • 无算法混淆攻击 — PASETO 在协议层面固定算法，不存在 JWT 的 "alg: none" 漏洞。
  • 更简洁的 API — 只有 create / parse 两个操作，无需手动处理 Header。
  • local 模式加密 — 令牌内容对称加密，payload 不明文暴露。
  • v4 协议 — 使用现代加密原语（AES-256-CTR + HMAC-SHA-384）。

令牌流程

  1. 用户登录 — 客户端发送 POST /api/auth/login，携带手机号与密码。
  2. 验证凭据 — 后端查询 users 表，校验密码哈希。
  3. 签发令牌 — 调用 paseto_handler.create_token(user_id, role)，生成包含 sub、role、iat、exp 声明的 v4.local 令牌。
  4. 返回令牌 — 响应体返回 { "access_token": "...", "token_type": "paseto" }。
  5. 存储令牌 — 客户端将令牌存入 localStorage（Web）或安全存储（Flutter）。
  6. 携带令牌 — 后续请求在 Header 中携带 Authorization: Bearer <token>。
  7. 验证令牌 — 后端通过 verify_token() 解密并校验过期时间，提取用户信息。

核心代码

# app/auth/paseto_handler.py
from datetime import datetime, timedelta, timezone
import paseto
from paseto.keys.symmetric_key import SymmetricKey
from paseto.protocols.v4 import ProtocolVersion4 as v4

def _get_key() -> SymmetricKey:
    key_bytes = settings.paseto_secret_key.encode()
    if len(key_bytes) < 32:
        key_bytes = key_bytes.ljust(32, b"\x00")
    return SymmetricKey(key_material=key_bytes[:32], protocol=v4)

def create_token(user_id: str, role: str) -> str:
    now = datetime.now(timezone.utc)
    exp = now + timedelta(minutes=settings.paseto_token_expire_minutes)
    payload = {"sub": user_id, "role": role, "iat": now.isoformat(), "exp": exp.isoformat()}
    return paseto.create(key=_get_key(), purpose="local", claims=payload)

def verify_token(token: str) -> dict | None:
    try:
        result = paseto.parse(key=_get_key(), purpose="local", token=token)
        return result["message"]
    except Exception:
        return None
安全提示

生产环境务必将 PASETO_SECRET_KEY 设置为 32 字节以上的强随机密钥，切勿使用默认值。密钥配置在 .env 文件中，不应提交到版本控制。

05WebSocket 实时通信

平台通过 WebSocket 实现多端实时协同，支持项目级消息广播。

连接端点

ws://host:port/ws/{project_id}

客户端通过项目 ID 建立 WebSocket 连接，同一项目的所有连接会被归组，实现消息广播。

消息格式

// 客户端发送
{ "event": "message", "data": { "text": "施工进度已更新" } }

// 服务端广播给同项目其他客户端
{ "event": "message", "data": { ... } }

// 错误消息
{ "event": "error", "data": { "message": "Invalid message format" } }

核心实现

WebSocket 管理器（app/ws.py）维护每个项目的连接池，支持 connect、disconnect、broadcast_to_project、send_to 方法。在 app/main.py 中注册 /ws/{project_id} 路由，接收 JSON 消息并按 event 分发广播。

Nginx 代理配置

生产环境需在 Nginx 中配置 WebSocket 升级：proxy_http_version 1.1 + Upgrade + Connection "upgrade"。详见部署指南。

06数据库迁移：Alembic

使用 Alembic 管理数据库 Schema 变更，支持 SQLite 与 PostgreSQL 双库。

常用命令

# 检测模型与数据库差异
alembic check

# 自动生成迁移脚本
alembic revision --autogenerate -m "add new table"

# 应用迁移到最新版本
alembic upgrade head

# 回滚一个版本
alembic downgrade -1

# 查看迁移历史
alembic history

迁移目录结构

alembic/
├── versions/
│   ├── 4356fec95e3e_init.py              # 初始化迁移
│   └── 8c945de89e0d_add_phase2_4_...py   # Phase 2-4 表迁移
├── env.py                                 # 迁移环境配置
└── script.py.mako                         # 迁移脚本模板
开发流程

修改 app/models/ 中的模型后，执行 alembic revision --autogenerate 生成迁移脚本，检查无误后 alembic upgrade head 应用。迁移脚本应纳入版本控制，确保团队数据库结构一致。

08部署：Nginx + uvicorn

09鸿蒙适配说明

Flutter App 已完成 HarmonyOS（鸿蒙）原生适配。

适配版本

  • Flutter — 3.35.7-ohos-0.0.3（鸿蒙定制版 Flutter）
  • HarmonyOS API — 23+（DevEco Studio）
  • 目标设备 — 华为 MatePad / MatePad Pro 等平板设备

适配要点

  • AR Engine — AR 空间测量通过 MethodChannel 调用鸿蒙原生 AR Engine，实现与 ARKit / ARCore 对等的扫描能力。
  • 手写笔 — stylus_adapter.dart 适配华为 M-Pencil，支持压感与手势绘制。
  • CustomPainter — 2D CAD 引擎基于 CustomPainter 实现，跨平台一致。
  • HAP 构建 — scripts/deploy-ohos.sh 一键构建 HAP 包并部署到 DevEco Studio。
